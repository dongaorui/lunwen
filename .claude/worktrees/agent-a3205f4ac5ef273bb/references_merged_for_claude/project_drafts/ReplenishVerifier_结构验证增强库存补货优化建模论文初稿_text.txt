基于 LP 结构验证的大语言模型库存补货优化建模方法研究
——一种面向低资源场景的 Solver-Guided ReplenishVerifier 框架
作者：XXX    单位：XXX    邮箱：XXX
摘要：大语言模型在自然语言到运筹优化模型的自动转化中表现出较强潜力，但在库存补货等高结构决策场景中，模型仍容易生成可执行但语义错误的数学规划模型，例如遗漏库存平衡、固定订货成本、Big-M 连接约束或缺货变量。仅依赖代码可执行性、求解器返回状态或目标值一致性，难以解释建模错误的结构来源。为此，本文提出 ReplenishVerifier，一种面向库存补货优化建模的 LP 结构验证增强框架。该框架首先执行大语言模型生成的求解器代码并导出 LP 文件，然后解析目标函数、变量、约束和二进制变量信息，进一步结合库存补货领域规则检查关键建模结构，最终用于候选选择、结构化反馈与自修复。本文构建了一个可验证的 ReplenishmentOR 原型数据集，覆盖单周期报童模型、多周期库存、允许缺货的补货模型、多品类容量约束模型和固定订货成本模型。实验设计表明，本文方法可与 Direct、CoT、Best-of-K、Solver-Filter 和通用 LP 统计方法进行系统比较，并通过结构完整率、库存平衡准确率、约束覆盖率和目标值准确率综合评估模型质量。本文工作为低资源条件下提升 LLM 运筹建模可靠性提供了一种可解释、可复现的思路。
关键词：大语言模型；库存补货；运筹优化建模；LP 结构验证；求解器反馈；自修复
Abstract: Large language models have shown promising capability in translating natural language descriptions into mathematical optimization models and solver code. However, in structured replenishment optimization scenarios, LLMs may produce executable but semantically incomplete formulations, such as missing inventory-balance constraints, setup binary variables, shortage variables, or Big-M linking constraints. This paper proposes ReplenishVerifier, an LP-structure-grounded verification framework for LLM-based replenishment optimization modeling. The framework executes generated solver code, exports LP files, parses variables, constraints and objective terms, and checks replenishment-specific modeling structures. The resulting structural signals can be used for candidate selection, feedback generation and self-repair. A prototype ReplenishmentOR benchmark is designed to evaluate executable rate, optimal rate, objective accuracy, structure completeness, inventory-balance accuracy and constraint coverage. The proposed framework provides an interpretable and low-resource approach for improving the reliability of LLM-based optimization modeling.
1 引言
库存补货优化是供应链管理、零售运营和生产计划中的基础问题。实际业务中，管理者需要根据需求、库存、提前期、容量、订货成本、持有成本和缺货惩罚等信息制定补货决策。传统方法通常要求建模人员将业务语言转化为线性规划、整数规划或混合整数规划模型，再调用求解器得到最优或近似最优方案。该流程对建模经验依赖较高，也限制了优化技术在非专业用户中的应用。
随着大语言模型的发展，利用 LLM 自动生成数学模型和求解器代码成为新的研究方向。已有工作从提示工程、监督微调、数据合成和可验证奖励等角度提升模型的自动运筹建模能力。例如，LLMOPT 通过五元素优化表达提升通用优化问题定义能力；OptMATH 从数学模型出发进行双向数据合成；OR-R1 将 SFT 与测试时强化学习相结合，并设计格式奖励、可执行代码奖励和多数投票奖励；SIRL 则进一步利用求解器执行结果与 LP 文件统计构造可验证奖励。
然而，现有方法大多关注结果层面的正确性，如代码能否运行、求解器是否返回可行或最优解、目标值是否与参考答案一致、多个候选答案是否投票一致。这些信号对筛选错误答案有帮助，但在库存补货等高结构问题中仍存在不足：一个模型即使可以被求解器执行，也可能遗漏库存平衡、容量约束或固定订货成本等关键结构，从而得到业务语义上错误的补货方案。
本文认为，库存补货优化问题具有清晰且可验证的领域结构。若问题描述中出现“每期库存由上一期库存、订货量与需求共同决定”，则模型应包含库存平衡；若出现固定订货成本，则模型通常需要二进制订货触发变量及 Big-M 连接约束；若出现缺货惩罚，则模型应包含缺货变量及相应目标函数项。因此，将这些领域结构映射为 LP 文件中的可检验模式，可以为 LLM 输出提供比结果正确性更细粒度的过程监督。
基于上述观察，本文提出 ReplenishVerifier 框架。该框架不直接替代 LLM，而是在 LLM 生成候选模型之后，通过“执行-导出-解析-验证-反馈”的流程检查其数学规划结构。相比普通 Solver-Filter，本文方法不仅判断代码是否可运行，还能说明模型缺少哪些关键补货结构；相比通用 LP 统计，本文方法进一步利用库存语义规则识别领域特定错误。
本文的主要贡献如下：第一，提出面向库存补货优化的可验证建模任务与 ReplenishmentOR 原型数据集设计，覆盖多种库存补货模型结构；第二，提出一种 LP 结构解析与补货语义规则结合的结构验证方法，能够检测库存平衡、订货变量、缺货变量、容量约束、二进制订货变量和 Big-M 约束等关键结构；第三，设计结构验证增强的候选选择与反馈机制，为低资源条件下提升 LLM 自动运筹建模可靠性提供可解释框架。
2 相关工作
2.1 大语言模型自动运筹建模
LLM 自动运筹建模旨在将自然语言问题描述转化为数学规划模型或可执行求解器代码。早期工作多依赖提示工程和少样本示例，后续研究逐渐转向学习型方法。LLMOPT 将优化问题抽象为统一的五元素表达，包括决策变量、目标函数、约束、参数和求解过程，用以增强模型对多类型优化问题的适配能力。OptMATH 则强调高质量优化建模数据的构造，从数学模型生成问题数据，再反向生成自然语言，并通过前向建模和拒绝采样保障数据一致性。
这些工作说明，优化建模能力不仅取决于语言理解，也取决于模型是否掌握数学规划的结构模式。库存补货问题虽然属于运筹学中的特定领域，但其模型结构清晰、可解释、可由求解器验证，适合作为研究 LLM 可靠建模能力的垂直场景。
2.2 求解器反馈与可验证奖励
OR-R1 和 SIRL 等工作表明，外部求解器可以为 LLM 生成的优化模型提供客观反馈。OR-R1 使用格式奖励、有效代码奖励和多数投票奖励，引导模型生成更规范、更可执行、更一致的解答。SIRL 将求解器返回的可行性、目标值以及 LP 文件中的模型统计信息作为可验证奖励信号，从而提升生成模型的真实性和可执行性。StepORLM 进一步强调生成式过程监督，通过过程奖励模型选择更优候选。
本文与这些工作一致，都认为求解器反馈是提升 LLM 运筹建模可靠性的关键。但不同之处在于，本文不满足于通用的执行结果或 LP 统计，而是针对库存补货问题进一步设计语义结构验证规则。换言之，本文关注的不是“代码是否能跑”，而是“生成的 LP 模型是否包含该类补货问题应有的数学结构”。
2.3 库存补货优化
库存补货优化包含一系列经典模型，如经济订货批量、报童模型、多周期库存模型、允许缺货的库存模型、多品类容量约束补货模型和固定订货成本模型。其共同特点是具有可解释的决策变量和约束结构：订货量控制补货动作，库存变量刻画状态转移，缺货变量刻画需求未满足部分，容量约束限制仓储或采购能力，二进制变量用于表示是否发生订货。
这些结构非常适合转化为验证规则。例如，多周期库存模型应包含跨时期库存平衡约束；固定订货成本模型应包含二进制变量及订货量与二进制变量之间的 Big-M 连接约束。本文正是利用这些结构先验，将库存补货知识转化为可执行、可解释的 LP 结构验证过程。
3 问题定义
给定一个库存补货优化问题的自然语言描述 x，LLM 需要生成数学建模过程和求解器代码 z。执行 z 后可得到优化模型 M、求解状态 s、目标值 v 以及导出的 LP 文件 l。传统评价通常比较 v 与参考目标值 v* 是否一致，或检查 s 是否为 Optimal。本文进一步定义结构正确性评价：若问题 x 中包含某类库存补货结构需求，则生成模型 M 中也应出现相应数学结构。
设 E(x) 表示由问题描述或参考模型得到的期望结构集合，例如库存平衡、容量约束、缺货变量、Big-M 约束等；设 D(l) 表示从 LP 文件中解析得到的实际结构集合。结构完整率可定义为：
StructureCompleteness(x, l) = |E(x) ∩ D(l)| / |E(x)|
本文任务目标是在候选集合 {z1, z2, ..., zk} 中选择结构最完整、执行最稳定、目标值最可信的候选，并在必要时生成结构化反馈以辅助模型修复。

4.2 LP 结构解析
LP 文件是求解器导出的标准化模型表示，包含目标函数、约束、变量边界和整数/二进制变量声明。本文解析 LP 文件中的四类信息：第一，目标函数项，用于判断是否包含持有成本、缺货成本和固定订货成本；第二，约束名称及其变量集合，用于识别库存平衡、容量约束和 Big-M 连接；第三，变量名称及其类型，用于识别订货变量、库存变量、缺货变量和二进制订货变量；第四，变量-约束关联关系，用于降低仅依赖变量命名导致的误判。
在实现上，系统既利用变量命名模式，如 Q、I、B、Y，也利用约束结构模式。例如，若某个约束同时包含相邻时期库存变量、订货变量和需求相关项，则可作为库存平衡约束的候选证据；若一个连续订货变量与二进制变量共同出现在形如 Q_t - M Y_t <= 0 的约束中，则可作为 Big-M 连接约束的证据。
4.3 补货语义规则
本文设计的补货语义规则包括十类：订货变量、库存变量、缺货变量、库存平衡约束、容量约束、二进制订货变量、Big-M 约束、持有成本目标项、缺货成本目标项和固定订货成本目标项。每条规则返回是否通过、置信度、证据和缺失说明。

4.4 综合评分与候选选择
为兼顾结果正确性和结构正确性，本文定义综合评分如下：
Score = 0.25 S_exec + 0.20 S_feas + 0.25 S_obj + 0.20 S_struct + 0.10 S_sem
其中 S_exec 表示代码是否可执行，S_feas 表示求解器是否返回可行或最优状态，S_obj 表示目标值是否接近参考目标值，S_struct 表示结构完整率，S_sem 表示自然语言语义与 LP 结构的一致性。若没有参考目标值，系统可降低目标值项权重，并提高结构完整率与可行性权重。
对于同一问题的多个候选，系统选择综合分数最高的候选作为最终输出。该过程与 Best-of-K 不同，后者通常仅依赖采样次数或多数投票；也不同于 Solver-Filter，后者主要依赖执行状态。ReplenishVerifier 的优势在于能够识别“能运行但结构不完整”的候选。
4.5 结构化反馈与自修复
当候选模型缺失关键结构时，系统会将验证失败项转化为自然语言反馈。例如，若问题中存在固定订货成本但 LP 中没有二进制变量，则反馈为：“题目包含固定订货成本，但你的模型没有二进制订货变量 Y[t]，应加入 Y[t] 并在目标函数中加入 fixed_cost * Y[t]。”若存在二进制变量但没有 Big-M 连接，则反馈为：“你的模型有订货触发变量，但缺少 Q[t] <= M * Y[t]，因此固定订货成本与订货量没有正确关联。”
这种反馈既可以在推理阶段用于自修复，也可以用于构造偏好学习数据。例如，将结构完整且目标值正确的候选作为 chosen，将可执行但缺失关键结构的候选作为 rejected，进而进行轻量 DPO 或 LoRA 微调。
5 实验设计
5.1 数据集与设置
实验拟采用两类数据。第一类为本文构造的 ReplenishmentOR 数据集，主要验证库存补货结构规则的有效性。第二类为公开运筹建模数据集中的库存、供应链、生产计划或容量规划相关子集，用于验证方法的外部适用性。由于本文定位为低资源方法，模型侧优先选择 Qwen2.5-3B、Qwen2.5-7B 和 Qwen3-8B 等可在 2×A40 级别设备上运行的开源模型。
5.2 对比方法
为公平评估本文方法，实验拟设置以下对比方法：（1）Direct，直接使用 LLM 第一次生成结果；（2）CoT，加入建模步骤提示；（3）Best-of-K，生成多个候选并选择可执行或多数一致候选；（4）Solver-Filter，仅依据代码可执行性、求解状态和目标值筛选；（5）Generic LP Stats，使用通用 LP 统计信息，如变量数、整数变量数、约束数量和目标方向；（6）Self-Repair，将执行错误反馈给模型进行修复；（7）ReplenishVerifier-Full，使用本文完整结构验证与评分。

6 案例分析
以一个含固定订货成本的多周期库存问题为例，题目要求每期可选择是否订货，若订货则产生固定费用，同时存在订货量、库存持有成本和需求满足约束。某些 LLM 生成的代码可能包含订货变量 Q[t] 和库存变量 I[t]，并能被求解器执行，但遗漏二进制变量 Y[t] 与 Big-M 连接约束 Q[t] <= M Y[t]。此时，普通 Solver-Filter 可能认为该候选可接受，因为代码能运行并返回目标值；但 ReplenishVerifier 会根据“固定订货成本”语义和 LP 中的变量结构发现错误，指出固定成本没有与订货行为绑定。
该案例说明，结构验证的价值不在于替代求解器，而在于补充求解器难以直接表达的业务语义。求解器只能求解给定模型，却不会判断该模型是否忠实表达了自然语言需求。通过 LP 结构验证，系统可以在建模层面对 LLM 输出进行细粒度检查，从而减少“可执行但错误”的风险。
7 讨论与局限性
本文方法仍存在若干局限。第一，结构规则依赖领域知识，若扩展到排班、路径规划或资源分配等问题，需要重新设计相应结构规则。第二，LP 结构解析可能受到变量命名和建模等价变换影响，因此后续应进一步引入变量-约束图匹配，以降低命名依赖。第三，本文框架需要执行候选代码并导出 LP 文件，运行成本高于直接生成。第四，若自然语言问题本身含糊不清，结构验证器也难以确定唯一正确结构。
未来工作可以从三个方向改进：其一，将库存规则抽象为通用优化结构验证语言，以支持更多运筹领域；其二，将 LP 结构证据注入过程奖励模型，形成结构增强的 GenPRM；其三，基于结构分数构造偏好对，对小模型进行低资源 DPO 或 GRPO 训练。
8 结论
本文提出 ReplenishVerifier，一种面向库存补货优化建模的 LP 结构验证增强框架。该框架通过执行 LLM 生成的求解器代码、导出并解析 LP 文件、结合库存补货语义规则检查关键结构，实现对“可执行但语义错误”模型的细粒度识别。相比仅使用代码执行或目标值一致性的筛选方法，本文方法更强调模型内部结构正确性，可为候选选择、自修复和偏好学习提供解释性过程监督。虽然本文仍处于原型设计阶段，但其为低资源条件下提升 LLM 自动运筹建模可靠性提供了一条可实施路径。
