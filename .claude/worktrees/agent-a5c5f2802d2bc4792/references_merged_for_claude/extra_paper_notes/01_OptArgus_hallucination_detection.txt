# OptArgus: A Multi-Agent System to Detect Hallucinations in LLM-based Optimization Modeling

URL: https://arxiv.org/abs/2605.11738
类别: 最接近：结构一致性/幻觉检测

## 为什么对 ReplenishVerifier 有帮助
这篇最容易被审稿人拿来和 ReplenishVerifier 比。它把 LLM 优化建模错误定义为 problem description、symbolic model、solver implementation 之间的 structural consistency 问题，并给出 objective/variable/constraint/implementation 错误分类。

## 你论文中怎么用
- Related Work 必须引用，不能说现有方法没有结构验证。
- 实验上可以做 OptArgus-like Audit baseline：只检查通用 objective/variable/constraint/implementation 错误，不使用库存补货语义规则。
- 差异点：ReplenishVerifier 面向 inventory replenishment，把 inventory balance、shortage、capacity、fixed cost、Big-M 等变成 LP-level structure signals，并用于 candidate selection、repair prompt、preference data。

## 风险 / 审稿人可能怎么问
如果 ReplenishVerifier 只做结构检测而没有 candidate selection / repair / preference data，它会显得像 OptArgus 的垂直弱化版。

## 给 Claude Code 的可执行指令
请实现 OptArgus-like Audit baseline：objective_present, variables_present, constraints_present, implementation_executable, solver_call_present, lp_export_present, solver_status_valid；输出 generic error types；不要使用 replenishment-specific rules。

## 建议放入论文的定位句
ReplenishVerifier should be positioned as a replenishment-specific LP-structure supervision layer that complements this line of work, rather than a replacement for it.
