# OptiMUS: Scalable Optimization Modeling with (MI)LP Solvers and Large Language Models

URL: https://arxiv.org/abs/2402.10172
类别: LLM-for-OR agent 框架

## 为什么对 ReplenishVerifier 有帮助
OptiMUS 是 LLM + solver + code generation/debugging 的代表。它提出 agent 能建模、写 solver code、debug、检查解，并发布 NLP4LP。

## 你论文中怎么用
- 作为 LLM 自动建模和 solver code debugging 代表工作。
- 提醒论文不要声称首次 solver-in-the-loop。
- 实验对比可用 Direct/Best-of-K/Solver-Filter 作为 OptiMUS-style lightweight baseline。

## 风险 / 审稿人可能怎么问
OptiMUS 是通用 agent 系统，若你的论文只做 PuLP 规则，系统完整性上比不过；必须强调你做的是 LP-level replenishment structure supervision。

## 给 Claude Code 的可执行指令
Related Work 中写：OptiMUS focuses on general modeling/debugging, while ReplenishVerifier focuses on domain-specific LP structure evidence for replenishment.

## 建议放入论文的定位句
ReplenishVerifier should be positioned as a replenishment-specific LP-structure supervision layer that complements this line of work, rather than a replacement for it.
